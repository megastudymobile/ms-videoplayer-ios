# 라이브 데모 스크립트 — 플레이어 모듈 리뷰 발표

> 작성: JunyoungJung · 2026-06-11
> 발표: 2026-06-12 전체 회의 (30~40분 + Q&A)
> 데모 슬롯: 발표 중반 "live demo" 슬라이드 시점, 약 8~10분

## 사전 준비 (발표 전날 체크리스트)

- [ ] 실기기(iPhone) 충전 + Wi-Fi 연결 확인 — Kollus DRM·다운로드는 시뮬레이터 불가
- [ ] `Example/Resources/kollus.local.plist` 자격증명 유효 확인
- [ ] `tuist generate` → Example 앱 Debug 빌드 → 실기기 설치
- [ ] 데모용 다운로드 콘텐츠 1개 **미리 다운로드 완료** 상태로 준비 (오프라인 시연용)
- [ ] 만료된(또는 만료 임박) 라이선스 콘텐츠 1개 준비 — 사전 검증 실패 시연용
- [ ] 화면 미러링 (QuickTime 유선 미러링 권장 — AirPlay는 회의실 네트워크 변수)
- [ ] **백업 영상 녹화**: 아래 데모 순서 전체를 사전 녹화, 로컬 보관
- [ ] 슬라이드 2종 브라우저 탭으로 열어두기 (`open docs/presentation/*.html`)

## 데모 순서

### 1. HLS 재생 — AVPlayerAdapter (약 2분)

1. Example 앱 메인 → 샘플 HLS 진입
2. 재생/일시정지/시킹 — 상태 스트림이 Observer 로그 뷰에 찍히는 것 강조
3. **멘트**: "화면은 `.play` 의도만 보내고, 상태는 reducer가 계산한 스트림으로 돌아옵니다."

### 2. 엔진 스왑 — Kollus DRM (약 2분)

1. 같은 플레이어 화면으로 Kollus 데모 콘텐츠 진입
2. **멘트**: "방금 화면과 코드가 같습니다. 바뀐 건 조립 지점의 엔진 하나."
3. Observer 뷰에서 DRM 콜백 로그 잠깐 노출 — "26개 델리게이트가 이 단일 스트림으로 정규화됩니다."

### 3. 다운로드 + 오프라인 (약 2~3분)

1. 다운로드 화면 → 진행률 스트림 시연
2. 미리 받아둔 콘텐츠 → 비행기 모드 → 오프라인 재생
3. 만료 콘텐츠 재생 시도 → **재생 전에** 실패 알림 뜨는 것 강조
4. **멘트**: "레거시는 틀어봐야 실패를 알았습니다. 지금은 시도 전에 압니다."

### 4. Skin 제스처 (약 2분)

1. 더블탭 좌/우 시킹, 롱프레스 2배속, 스크럽 중 썸네일 프리뷰
2. 스크린샷 시도 → secure 캔버스로 검은 화면 캡처되는 것 시연
3. **멘트**: "이 UI 전부 블록 조립식 — host가 슬롯 단위로 끼우고 뺍니다."

## 실패 대응 (백업 플랜)

| 상황 | 대응 |
|------|------|
| Wi-Fi/네트워크 불안정 | 오프라인 데모(3번)부터 시작, 스트리밍은 녹화 영상으로 |
| 미러링 끊김 | QuickTime 유선 재연결 1회 시도 → 안 되면 녹화 영상 |
| DRM 라이선스 서버 이슈 | 즉시 녹화 영상 전환 — "실기기 검증 환경 의존" 멘트로 자연스럽게 |
| 앱 크래시 | 재실행 1회 → 안 되면 녹화 영상. 크래시 자체를 숨기지 말 것 (리뷰 회의임) |

## 데모 후 연결 멘트

데모 끝나면 "품질 가드" 슬라이드로 복귀:
"방금 보신 동작 뒤에는 232개 테스트와 경계 테스트가 있습니다. 이제 아직 안 되는 것들을 말씀드리겠습니다."
